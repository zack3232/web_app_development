# PRD - 食譜收藏夾系統

## 1. 專案概述

- **背景與動機**：現代人常在網路上看到各種食譜，但散落在不同平台（FB、IG、部落格），難以統一管理。當實際要煮飯時，往往找不到當初收藏的食譜。本系統旨在提供一個統一集中的平台，讓使用者能輕鬆收藏、整理及尋找愛用的食譜。
- **目標用戶**：喜歡在家烹飪的自煮族群、家庭主婦/夫，以及需要管理大量食譜的料理愛好者。
- **核心價值主張**：集中管理、快速查找，讓每次下廚不再手忙腳亂。

## 2. 功能需求

以下為本系統的主要功能與對應的使用者故事：

1. **會員帳戶管理（註冊、登入與登出）**
   - 作為「一般使用者」，我希望「能註冊並登入專屬帳號」，以便「擁有我個人的私密食譜收藏空間」。

2. **新增與編輯食譜**
   - 作為「料理愛好者」，我希望「能手動新增或編輯食譜（包含食材、步驟與圖片）」，以便「記錄我專屬的美味秘訣」。

3. **食譜分類與標籤**
   - 作為「自煮族群」，我希望「能為食譜加上標籤（如：中式、甜點、減脂）與自訂分類」，以便「未來能根據想吃的類別快速找到對應的料理」。

4. **食譜搜尋與篩選**
   - 作為「家庭主婦/夫」，我希望「能透過關鍵字搜尋或標籤篩選來尋找食譜」，以便「在不知道煮什麼時，能快速找到適合的靈感」。

5. **我的最愛清單**
   - 作為「一般使用者」，我希望「能將特定的食譜加入『我的最愛』」，以便「能立刻查看最近想嘗試或最常煮的幾道菜」。

## 3. 非功能需求

- **技術限制**：
  - 後端技術：使用 Python 的 Flask 框架
  - 前端頁面：使用 Jinja2 模板引擎搭配原生 HTML/CSS/JavaScript
  - 資料庫：使用 SQLite 管理關聯性資料表
- **效能與安全考量**：
  - 使用者的密碼需經過雜湊處理避免明文儲存。
  - 對重要的寫入操作需經過身份驗證及權限檢查（如：僅能編輯與刪除自己建立的食譜）。
  - 對使用者輸入進行防護與過濾，防止 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍

- **Must Have（必須具備）**：
  - 註冊系統與使用者登入登出機制
  - 新增、修改、刪除、瀏覽食譜（CRUD 核心功能）
  - 全部食譜列表呈現與個人專屬食譜呈現
- **Should Have（應該具備）**：
  - 搜尋功能（依據標題或食材搜尋）
  - 標籤或簡易的分類功能
  - 加入「我的最愛」功能
- **Nice to Have（最好具備）**：
  - 貼上網址自動匯入網頁食譜大綱
  - 將食譜自動列出對應的採買清單功能
  - 隨機推薦（不知道煮什麼的時候）
